搭建博客过程

发表于 2017-04-12 | 分类于 技术

前言

github为广大开发者提供了代码托管平台，甚至可以托管静态博客。考虑到github在很长的时间内都会运营，所以开发者几乎可以零成本建立一个属于自己的静态博客。

Hexo

静态博客可供选择的也有很多，目前比较热门的有Jekyll，Hexo等。我发现Hexo的使用简单，主题丰富，所以我选择了用Hexo搭建静态博客。
Hexo有一下特点：

  • 超快速度：Node.js 所带来的超快生成速度，让上百个页面在几秒内瞬间完成渲染。
  • 支持Markdown： Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至可以整合 Octopress 的大多数插件。
  • 一键部署：只需一条指令即可部署到 GitHub Pages, Heroku 或其他网站。
  • 丰富的插件：Hexo 拥有强大的插件系统，安装插件可以让 Hexo 支持 Jade, CoffeeScript。

  • 选择Scheme
    Scheme 是 NexT 提供的一种特性，借助于 Scheme，NexT 为你提供多种不同的外观。同时，几乎所有的配置都可以 在 Scheme 之间共用。目前 NexT 支持三种 Scheme，他们是：

    • Muse - 默认 Scheme，这是 NexT 最初的版本，黑白主调，大量留白
    • Mist - Muse 的紧凑版本，整洁有序的单栏外观
    • Pisces - 双栏 Scheme，小家碧玉似的清新
      Scheme 的切换通过更改主题配置文件，搜索 scheme 关键字。 你会看到有三行 scheme 的配置，将你需用启用的 scheme 前面注释 # 去除即可。

  • 菜单

还需要设置菜单项的图标，对应的字段是menu_icons。此设定格式是item name: con name，其中 item name 与上一步所配置的菜单名字对应，icon name 是 Font Awesome 图标的 名字。而 enable 可用于控制是否显示图标，你可以设置成 false 来去掉图标。

绑定域名

虽然github提供的域名已经足够简单记忆了，但是毕竟还是github的二级域名，仍旧显得不够高大上。所以，我们可以注册一个域名，并让那个域名直接映射到github上的博客地址。
在上篇博文中，我已经提到我购买了本博客的域名kazuyayx.com。接下来就介绍一下如何绑定域名至github的博客上。
由于域名是从godaddy上购买的，而godaddy的域名解析服务器时常会被墙，所以采用国内腾讯云的域名解析服务dnspod。

  1. 在source文件夹中新建一个CNAME文件(无后缀名)，然后用文本编辑器打开，在首行添加你的网站域名，如http://xxxx.com，注意前面没有http://，也没有www，然后使用hexo g && hexo d上传部署。

总结

其实，搭建一个博客并不是太难，毕竟现在成熟的框架已经太多太多，完全不需要自己再造轮子。当然，如果对前端感兴趣的人或许可以自己动手改改前端样式，做一个属于自己的主题。总而言之，我今后一定要克服对新技术的恐惧。很多事情并不是很难，只是我们尚未接触罢了！
